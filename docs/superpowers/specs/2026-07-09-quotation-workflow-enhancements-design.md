# 报价工作流体验优化设计

## 目标

完善核算、报价单、物流管理、报价记录和 PDF 导出之间的数据流与操作反馈，同时安全兼容已有本地记录和当前尚未提交的装箱明细功能。

## 总体方案

采用结构化数据模型升级，并在读取旧数据时做兼容迁移。迁移只补充新字段、转换旧物流总价，不删除未知字段，也不覆盖用户原始内容。所有数据仍保存在浏览器本地 IndexedDB 或 localStorage 中，继续支持现有 JSON 备份与恢复。

## 1. 核算页产品信息

核算页每个产品增加三个独立字段：

- 产品描述
- 型号 / 规格
- 材质 / 表面处理

生成报价单草稿、保存核算记录、重新打开核算记录时均保存和恢复三个字段。生成报价单时分别映射到报价单产品的 `desc`、`spec`、`material`，不再把型号拼接到描述中。旧核算记录只有产品名称时，将该值作为产品描述，型号和材质保持为空。

## 2. Declaration 显示规则

Quotation、Proforma Invoice 和 Commercial Invoice 使用同一规则：

- 填写 Declaration 时，在单据预览、打印/PDF 和 Excel 中显示。
- Declaration 为空或只包含空格时，不显示标题和空白区域。
- Packing List 不显示 Declaration。

## 3. 物流费用明细

每个物流方案保存结构化费用数组 `costItems`。默认费用项为：

- 海运费
- 港杂费
- 文件费
- 报关费

用户可以新增、改名和删除其他费用项。每项包含唯一 ID、费用名称和非负金额。费用币种沿用物流方案币种。

`totalCost` 改为只读自动汇总值，等于全部费用项金额之和。旧记录迁移规则：

- 如旧记录只有 `totalCost`，迁移为一项“原物流总成本”。
- 如旧记录只有 `unitPrice` 或最低收费，这些字段继续保留，用于计费参考，不自动计入费用明细。
- 未知旧字段原样保留，确保备份可以往返。

列表和搜索覆盖方案名称、国家、港口、供应商、运输方式、费用项名称和备注。

## 4. 报价单底部联系信息

单据底部只读取以下卖方字段：

- 卖方网址
- 卖方邮箱
- 卖方电话 / WhatsApp

有值的项目使用分隔符连接；三项都为空时，整个底部联系信息节点隐藏且不占高度。买方邮箱不会作为底部信息的回退值。Excel 导出使用相同规则。

## 5. 报价记录分类与备注

报价记录新增：

- `folderId`：所属分类文件夹
- `saveRemark`：保存备注

增加独立的文件夹集合，文件夹包含 ID、名称、创建时间和更新时间。系统提供“未分类”虚拟分组，旧记录默认进入未分类。

点击“保存”或“另存”时打开保存对话框，用户可以：

- 选择已有文件夹
- 新建文件夹
- 填写或修改保存备注
- 确认保存或取消

编辑已有记录时默认带出原文件夹和备注；“另存”创建新记录但继承当前选择，用户可以调整。记录页面支持：

- 按文件夹筛选
- 搜索客户、单号、产品、文件夹名称和保存备注
- 在记录卡片中显示文件夹和备注
- 新建、重命名和删除文件夹

删除非空文件夹时，其中记录移入“未分类”，不会删除报价记录。分类和备注加入 JSON 备份与恢复。

## 6. 保存和 PDF 导出反馈

所有保存入口统一返回明确结果：

- 保存成功：显示记录编号和成功状态。
- 保存失败：显示可读错误，保留当前表单数据。
- 用户取消保存对话框：不保存，不显示失败。

点击 PDF 时先执行记录保存：

- 保存失败或用户取消时，停止 PDF 导出。
- 保存成功后再调用浏览器打印窗口。
- 打印调用抛出异常时显示“PDF 导出窗口打开失败”及错误信息。
- 打印窗口正常返回后提示“打印窗口已关闭；是否保存文件取决于打印窗口中的操作”，不误报文件已经成功写入磁盘。

网页无法可靠获知浏览器打印窗口中用户点击了“保存”还是“取消”，因此不显示虚假的“PDF 已保存成功”状态。

## 数据兼容与迁移

迁移在数据读取层执行，并保持幂等：同一记录重复读取不会重复添加费用项或修改时间。新字段全部提供默认值。保存后使用新结构，读取时仍接受旧结构。

备份导出包含报价记录、记录文件夹、核算草稿、物流费用明细及现有其他数据。恢复时验证数组和对象形状；无效的新字段回退到默认值，不阻断其他有效数据恢复。

## 错误处理

- localStorage、IndexedDB 或备份恢复异常均显示在当前页面状态区。
- 金额输入限制为非负有限数字，空值按零处理。
- 删除文件夹、删除费用项等破坏性操作需要确认。
- 对话框可通过取消按钮和 Escape 关闭，关闭时不写入数据。

## 测试与验收

至少覆盖以下回归场景：

1. 核算页填写描述、型号和材质，生成报价单后字段保持独立且完整。
2. 旧核算记录仍能打开并生成报价单。
3. Declaration 在 Quotation、PI、CI 中有值显示、空值隐藏，在 PL 中始终隐藏。
4. 四项默认物流费用和自定义费用能够增删改，总成本实时准确汇总。
5. 旧物流记录迁移后总成本不变，重复读取不重复生成费用项。
6. 卖方网址、邮箱和电话全空时页脚无联系信息；买方邮箱不会出现在页脚。
7. 保存和另存能够选择文件夹并填写备注，记录页能筛选和搜索。
8. 删除非空文件夹后记录进入未分类。
9. 保存失败时 PDF 不打开并显示错误；打印调用失败时显示错误。
10. 新旧结构均可备份和恢复。
11. 当前装箱明细字段、Packing List 预览和 Excel 导出不发生回归。

## 非目标

- 不接入服务器、云同步或多用户权限。
- 不绕过浏览器打印对话框直接生成二进制 PDF。
- 不自动删除或合并用户已有记录。
